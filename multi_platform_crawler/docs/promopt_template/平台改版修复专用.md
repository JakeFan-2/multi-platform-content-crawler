## AI 提示词模板（平台改版修复专用）

### 使用说明
- **适用场景**：当采集系统中某个平台因页面改版导致登录失败、无法提取文章列表、字段缺失时使用。
- **操作方式**：将以下模板复制到 AI 编程工具（Cursor / Cline / Continue）的对话框中，按提示补充 `[平台标识]` 和 `[故障现象]`，然后发送给 AI。
- **前置条件**：AI 工具已配置 Playwright MCP 服务器，项目目录已打开。

---

### 模板内容

```markdown
# 任务：修复平台改版导致的采集失败

## 1. 项目背景（请严格遵循）
你是本项目的 AI 开发助手，项目为“多平台内容采集系统”（Python + PySide6 + Playwright）。请先阅读并理解以下两份核心文档：
- `/docs/项目总体方案策略描述文档_v3.0.md`（或项目中对应的文档路径）
- `/docs/平台改版应对方案_v1.1.md`

核心设计原则：
- 所有平台配置存放在 `platforms/{platform}.yaml`，修改配置即可适配改版，**不修改采集器代码**。
- 选择器使用多级 Fallback（id > css > role > text > xpath），保留原有选择器作为备份。
- 登录逻辑区分：
  - 微信公众号、企鹅号、雪球号：GUI 自动采集模式下 Cookie 失效时进入手动登录队列，不自动填表。
  - 其余平台：支持自动填表登录（账号密码来自 YAML 或 .env）。
- 曝光量字段不从网页提取，由 `config/exposure.yaml` 静态注入，**不要修改曝光量相关配置**。
- 修改 YAML 时保留所有注释和其他字段（超时、UA、翻页等），只替换失效的选择器。

## 2. 任务输入
- **平台标识**：[请填写，例如 toutiao / weibo / zhihu / qq / baijiahao / netease / yidian / zaker / xueqiu / wechat]
- **故障现象**：[请简要描述，例如“登录失败”、“采集到0篇文章”、“阅读量字段为空”]
- **目标页面 URL**（如已知）：[可选，若不提供，AI 将从 YAML 中读取 `login_url` 和 `article_list` 相关 URL]

## 3. 执行步骤（AI 必须按顺序执行）

### 3.1 读取当前配置
- 使用 `read_file` 读取 `platforms/{platform}.yaml`
- 了解现有的选择器结构，特别是 `login`、`article_list.row_selectors`、`article_list.field_selectors`

### 3.2 启动 Playwright MCP 会话
- 调用 `browser_navigate` 打开平台的登录页面（或直接打开需要分析的数据页面，若已登录可跳过）。
- 如果页面需要登录且当前未登录：
  - 对于支持自动填表的平台：尝试使用现有配置中的账号密码选择器进行登录（不修改登录流程，只观察页面结构）。
  - 对于微信公众号/企鹅号/雪球号：提示用户需要手动扫码，然后等待用户完成（MCP 可保持浏览器打开）。
- 登录成功后，导航到文章列表页（数据页面）。

### 3.3 分析页面结构
- 调用 `browser_snapshot` 获取页面的可访问树（包含 role、name、value 等）。
- 识别以下关键元素（根据故障现象决定重点）：
  - **文章行容器**：每个文章项的最外层元素（通常是 `<tr>`、`<div>`、`<li>`）。
  - **标题字段**：文章标题所在元素。
  - **阅读量、点赞量、评论量、转发量、收藏量**：对应数据字段。
  - **发布时间**（如果采集需要）。
  - **翻页按钮**（如果原配置有翻页且失效）。
- 对于每个元素，提取 **至少 3 个备选选择器**，优先级：`id` > `css` > `role` > `text` > `xpath`。

### 3.4 验证选择器有效性
- 使用 `browser_evaluate` 测试每个新选择器，确保能定位到元素并获取到非空值。
- 示例验证表达式：
  ```javascript
  document.querySelectorAll('新选择器').length
  ```
  或提取第一个元素的文本内容。

### 3.5 生成 YAML 更新建议
- 输出一份清晰的对比表格，展示：
  - 原配置中的失效选择器 → 建议的新选择器（多级 Fallback）
- **只修改失效的部分**，其他字段保持原样。
- 对于 `row_selectors`，采用**追加**方式：将新选择器放在数组最前面，保留旧的作为 fallback。

### 3.6 更新配置文件（需用户确认）
- 先备份原文件：`platforms/{platform}.yaml.bak`（使用 `write_file` 或 `execute_command`）。
- 使用 `replace_in_file` 或 `edit_file` 精确替换目标选择器数组。
- 更新后，提示用户重启程序并测试。

## 4. 输出要求
- 以结构化 Markdown 格式输出：
  - **问题定位**：说明哪个选择器失效，当前页面实际结构。
  - **新选择器建议**：按优先级列出。
  - **配置文件修改 diff**（仅显示改动部分）。
  - **验证步骤**：如何测试修改是否成功。
- 如果遇到无法自动解决的问题（如需要人工扫码登录、页面加载异常），明确提示用户手动干预。

## 5. 注意事项
- 不要修改 `config/exposure.yaml`。
- 不要修改 `collectors/*.py` 中的 Python 代码。
- 不要删除原配置文件中的任何字段，只替换或追加选择器。
- 对于微信公众号、企鹅号、雪球号，如果登录选择器失效，应引导用户手动扫码登录，而不是尝试自动填表。
- 如果页面结构变化巨大（例如从表格变成卡片流），可能需要调整 `article_list.iframe_selectors` 或 `pagination` 逻辑，此时应询问用户是否允许更大范围的修改。

## 6. 开始执行
请根据上述步骤，分析平台 `[平台标识]`，修复故障 `[故障现象]`，并输出修改建议。如有不确定的地方，请先提问。
```

### 使用示例（填充后）

```markdown
# 任务：修复平台改版导致的采集失败
...
## 2. 任务输入
- **平台标识**：toutiao
- **故障现象**：采集到0篇文章，日志提示“未找到文章行元素”
- **目标页面 URL**：https://mp.toutiao.com/profile_v4/graphic/articles
...
```

将上述完整内容发送给 AI 即可。AI 会按照模板中的步骤自动执行分析并更新配置。
