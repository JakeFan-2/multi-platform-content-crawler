# 任务：为系统新增一个内容平台采集器（全自动构建）

## 1. 项目背景（请严格遵循）
你是本项目的 AI 开发助手，项目为“多平台内容采集系统”（Python + PySide6 + Playwright）。请先阅读并理解以下核心文档：
- `/docs/项目总体方案策略描述文档_v3.0.md`（或项目中对应的文档路径）
- `/docs/单平台采集器构建策略文档_v2.0.md`（即本文档）

核心设计原则：
- **配置驱动**：所有平台相关选择器、URL、超时、重试均存放在 `platforms/{platform}.yaml`，采集器代码只实现通用逻辑。
- **无侵入扩展**：新增采集器不修改任何已有代码，只需添加新的 YAML 和 Python 文件。
- **模板继承**：所有采集器必须继承 `BaseCollector`（来自 `collectors/template_collector.py`），实现标准接口。
- **健壮性**：每个元素必须提供至少 3 个备选选择器（id > css > role > text > xpath），支持 Fallback。
- **安全合规**：Cookie 加密存储，敏感信息环境变量化。
- **曝光量不采集**：曝光量字段由 `config/exposure.yaml` 静态注入，采集器无需提取。

## 2. 任务输入（用户必须提供）
请填写以下最小信息，AI 将通过 Playwright MCP 自动分析并生成完整采集器：

- **平台标识**（唯一，小写+下划线）：`[例如: new_platform]`
- **显示名称**（GUI 显示）：`[例如: 新平台名称]`
- **登录 URL**：`[例如: https://example.com/login]`
- **数据后台 URL**（文章列表页）：`[例如: https://example.com/manager/articles]`
- **登录方式**（二选一）：
  - [ ] 账号密码登录（需提供测试账号密码，可留空从 .env 读取）
  - [ ] 扫码登录（如微信公众号、企鹅号等，采集器需支持手动扫码）
- **是否需要验证码**：是 / 否（如是，将暂停自动登录，提示用户手动输入）
- **文章列表形式**：
  - [ ] 表格（table/tbody/tr）
  - [ ] 列表（div/list-item）
  - [ ] 卡片流（无限滚动）
  - [ ] 其他：`[请描述]`
- **翻页方式**：
  - [ ] “下一页”按钮
  - [ ] 页码跳转
  - [ ] 无限滚动（需配置最大滚动次数）
  - [ ] 无翻页（单页）
- **每页文章数（大约）**：`[例如: 20]`（用于计算 `max_pages` 保证总文章约 60 篇）

## 3. 执行步骤（AI 必须严格按照策略文档 v2.0 执行）

### 3.1 环境准备与页面启动
- 调用 `browser_navigate` 打开登录 URL。
- 等待页面加载完成（`wait_until="networkidle"`）。
- 截图保存为 `snapshots/{platform}_login_page.png`。

### 3.2 登录页面分析（提取登录选择器）
- 调用 `browser_snapshot` 获取页面可访问树。
- 提取以下元素的多选择器（至少 3 个，按优先级）：
  - 用户名/账号输入框
  - 密码输入框
  - 登录按钮
  - 验证码输入框（如果有）
- 保存截图 `snapshots/{platform}_login_analysis.png`。

### 3.3 执行登录
- 根据用户指定的登录方式：
  - **账号密码登录**：使用提取的选择器自动填表并点击登录。如遇验证码，暂停并提示用户手动输入。
  - **扫码登录**：提示用户手动扫码，等待用户完成（设置超时 5 分钟）。
- 登录成功后：
  - 调用 `browser_snapshot` 验证登录态（查找登录后独有的元素，如“发布管理”）。
  - 保存 Cookie 到本地（后续用于 `LoginManager`）。
  - 截图 `snapshots/{platform}_logged_in.png`。

### 3.4 导航到数据后台页面
- 调用 `browser_navigate` 跳转到数据后台 URL。
- 等待文章列表加载完成（`wait_until="networkidle"`）。
- 截图 `snapshots/{platform}_data_page.png`。

### 3.5 文章列表结构分析（提取列表与字段选择器）
- 调用 `browser_snapshot` 获取数据页面结构。
- 识别：
  - **列表容器**（table、div.list、tbody 等）
  - **每一行的选择器**（tr、.article-item、[data-id] 等）
- 提取每个数据字段的选择器（至少 2 个备选）：
  - `title`（文章标题）
  - `url`（文章链接，通常从标题的 `href` 提取）
  - `publish_time`（发布日期，格式化为 YYYY-MM-DD）
  - `read`（阅读量）
  - `like`（点赞量）
  - `comment`（评论量）
  - `forward`（转发量）
  - `collect`（收藏量）
- 注意：某些字段可能不存在，在配置中留空或设为 `null`，调度层会自动填充 `/`。

### 3.6 翻页机制识别
- 检查是否有“下一页”按钮、页码控件或 URL 模式。
- 提取翻页选择器（至少 2 个）。
- 计算 `max_pages`：保证总采集文章数约 60 篇（例如每页 20 篇，则 `max_pages=3`）。

### 3.7 选择器测试与验证
- 使用 `browser_evaluate` 测试每个选择器，确保能定位到元素并获取非空值。
- 记录测试结果（成功/警告/失败），失败的选择器降级使用备用。

### 3.8 生成配置文件（YAML）
- 基于 `platforms/template.yaml`（或已有平台的 YAML 结构）生成 `platforms/{platform}.yaml`。
- 必须包含以下配置段：`platform`、`login_url`、`data_page_url`、`cookie_file`、`timeout`、`user_agents`、`login`、`article_list`（含 `row_selectors`、`field_selectors`、`pagination`）。
- 对于扫码登录的平台，`login` 部分只保留 `logged_in_indicator_selectors`，不设置自动填表选择器。
- 保留注释，格式规范。

### 3.9 生成采集器代码（Python）
- 基于 `collectors/template_collector.py` 生成 `collectors/{platform}.py`。
- 类名：`{PlatformName}Collector`（首字母大写驼峰）。
- 必须实现：
  - `PLATFORM_NAME = "{platform}"`
  - `is_logged_in(page)`：使用配置中的 `logged_in_indicator_selectors` 判断。
  - `perform_login(page, context)`：根据登录方式实现自动填表或等待手动扫码。
  - `navigate_to_article_list(page)`：导航到数据后台。
  - `_extract_page_articles(page)`：提取当前页文章列表，使用配置中的选择器。
  - `_go_to_next_page(page)`：翻页逻辑。
- 使用 `self.logger` 记录日志，关键操作添加重试装饰器 `@self.retry_mgr.async_retry_on_failure`。
- 不修改基类 `BaseCollector` 的任何方法签名。

### 3.10 输出分析报告与集成指引
输出报告反馈，不需要单独生成文件，包含：
- 页面分析摘要（iframe、动态加载、反爬特征）
- 提取的选择器列表（标注优先级和测试结果）
- 截图路径列表
- 集成步骤（如何注册到 `PlatformRegistry`、如何测试）
- 注意事项（如特殊单位转换、验证码处理、登录态保持策略）

## 4. 输出要求
- 以结构化 Markdown 格式输出最终报告。
- 在报告中提供配置文件 `platforms/{platform}.yaml` 和采集器代码 `collectors/{platform}.py` 的完整内容（可直接复制保存）。
- 若遇到无法自动解决的问题（如登录页面有复杂的验证码、动态 token 等），明确输出“需要人工介入”并说明原因。

## 5. 约束与禁止事项
- **禁止修改** `collectors/` 下任何已有平台的代码。
- **禁止修改** `platforms/` 下任何已有平台的 YAML 配置。
- **禁止修改** `template_collector.py` 基类。
- **禁止删除**原有配置文件中的任何字段，只追加新选择器。
- **禁止**在采集器代码中硬编码选择器（所有选择器必须从 YAML 读取）。
- **禁止**在采集器中处理曝光量字段（该字段由调度层注入）。
