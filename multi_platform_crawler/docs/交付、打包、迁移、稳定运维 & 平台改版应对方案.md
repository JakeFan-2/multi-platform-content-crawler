## 1. 概述与核心原则

### 1.1 方案定位
本文档定义如何将现有的多平台内容采集系统（Python + PySide6 + Playwright）**打包为单 exe 可交付物**，并确保**任意迁移、零配置启动、长期稳定运行**，同时支持**非技术人员在平台改版时仅通过修改外置文件恢复采集**。

### 1.2 核心原则（直接落地）
| 原则 | 说明 |
|------|------|
| **完整保留原始目录结构** | 不拆分、不移动、不重命名任何文件夹，开发期与打包后目录关系完全一致 |
| **动态根目录** | 全局仅使用 `PROJECT_ROOT = Path(sys.executable).parent if getattr(sys,'frozen',False) else Path.cwd()`，所有路径基于此拼接 |
| **硬编码禁止** | 代码中不得出现任何绝对路径（如 `C:/crawler/...`），全部动态派生 |
| **固定代码全打包** | `main.py`, `gui.py`, `controller.py`, `thread.py`, `utils/` 等永远不会随网页改版的代码全部打包进 exe |
| **易变模块外置** | 仅 `collectors/`（采集器代码）、`platforms/`（YAML 配置）、`.env`（账号密码配置）外置，允许运维人员直接修改 |
| **整文件夹迁移** | 交付物为一个文件夹，用户复制整个文件夹到任意位置/电脑，双击 exe 即可运行，无需任何配置 |

### 2.2 文件归属详细清单

| 类型 | 目录/文件 | 是否打包进 exe | 理由 |
|------|-----------|----------------|------|
| **可执行代码** | `main.py`, `gui.py`, `controller.py`, `thread.py`, `platform_registry.py`, `utils/*.py` | ✅ 全部打包 | 永不随网页改版变动，打包后文件夹更整洁 |
| **采集器实现** | `collectors/*.py` | ❌ 外置 | 随平台改版可能需要修改采集逻辑 |
| **平台配置** | `platforms/*.yaml` | ❌ 外置 | 随平台改版需要修改选择器 |
| **账号密码配置** | `.env` | ❌ 外置 | 统一存所有平台账号密码、飞书接口，外置不打包，安全易维护 |
| **运维配置** | `config/exposure.yaml` | ❌ 外置 | 曝光量数值可能频繁调整 |
| **静态资源** | `resources/icon.ico` | ✅ 打包（通过 `--add-data`） | 固定不变 |
| **运行时数据** | `cookies/`, `data/`, `logs/`, `snapshots/` | ❌ 不打包，运行时自动创建 | 用户数据，不能内置 |

> **注意**：`config/exposure.yaml` 虽然属于配置，但它是**运维可变**的，故外置。其他如日志格式、超时阈值等固定参数仍然硬编码或内置在 `utils/ops.py` 中。

## 3. 动态路径处理实现（关键代码）

### 3.1 获取项目根目录（通用函数）
在 `utils/path_helper.py`（新增文件）中实现：

```python
import sys
from pathlib import Path

def get_project_root() -> Path:
    """返回项目根目录（开发环境返回当前工作目录，打包环境返回exe所在目录）"""
    if getattr(sys, 'frozen', False):
        # 打包后运行：sys.executable 是 exe 的完整路径
        return Path(sys.executable).parent
    else:
        # 开发环境：返回当前工作目录（通常是项目根目录）
        return Path.cwd()

# 全局常量，供其他模块导入
PROJECT_ROOT = get_project_root()
```

### 3.2 所有子目录动态拼接
在需要访问外置目录或运行时目录的地方，统一使用：

```python
from utils.path_helper import PROJECT_ROOT

COLLECTORS_DIR = PROJECT_ROOT / "collectors"
PLATFORMS_DIR = PROJECT_ROOT / "platforms"
CONFIG_DIR = PROJECT_ROOT / "config"
COOKIES_DIR = PROJECT_ROOT / "cookies"
DATA_DIR = PROJECT_ROOT / "data"
LOGS_DIR = PROJECT_ROOT / "logs"
SNAPSHOTS_DIR = PROJECT_ROOT / "snapshots"
ENV_PATH = PROJECT_ROOT / ".env"
```

### 3.3 运行时目录自动创建
在主程序启动时（`main.py` 或 `gui.py` 初始化阶段）调用：

```python
def ensure_runtime_dirs():
    for d in [COOKIES_DIR, DATA_DIR, LOGS_DIR, SNAPSHOTS_DIR]:
        d.mkdir(exist_ok=True)
```

### 3.4 外置配置文件加载示例
```python

if exposure_path.exists():
    with open(exposure_path, 'r', encoding='utf-8') as f:
        exposure_config = yaml.safe_load(f)
else:
    # 可选：内置默认值或报错
    exposure_config = {}
```

## 4. 打包方案（PyInstaller 实施）

### 4.1 打包环境准备
- 安装 Python 3.10+（推荐 3.10）
- 安装项目依赖：`pip install -r requirements.txt`
- 安装 PyInstaller：`pip install pyinstaller`

### 4.2 打包命令（支持动态路径）
由于我们已将 `config/` 中的 `exposure.yaml`、`.env` 外置，且 `collectors/` 和 `platforms/` 外置，因此打包时**不需要**将这些目录打包进 exe。但需要将 `utils/` 等内部模块打包。另外，需要将 `resources/icon.ico` 作为资源加入。

在项目根目录（与 `main.py` 同级）执行：

```bash
pyinstaller --onefile --windowed ^
    --name "多平台内容采集系统" ^
    --icon=resources/icon.ico ^
    --add-data "resources/icon.ico;resources" ^
    --hidden-import=playwright.async_api ^
    --hidden-import=playwright._impl._api_types ^
    --hidden-import=asyncio ^
    --hidden-import=multiprocessing ^
    --collect-all=playwright ^
    --collect-all=playwright_stealth ^
    main.py
```

**参数解释**：
- `--onefile`：生成单个 exe
- `--windowed`：不显示控制台窗口（适合 GUI 程序）
- `--add-data`：将图标文件打包进 exe 内的 `resources` 目录（运行时可通过 `sys._MEIPASS` 访问）
- `--hidden-import`：确保 Playwright 异步模块被显式包含
- `--collect-all`：将整个 Playwright 及其依赖的包数据（如浏览器驱动）收集进 exe

> **注意**：Playwright 的浏览器二进制文件**不会**被打包进 exe（体积过大），首次运行时会自动下载到用户目录（`~/.cache/ms-playwright`）。这是合理行为。

### 4.3 打包后文件结构（exe 解压后内存视图）
当用户双击 exe 时，PyInstaller 会在临时目录（如 `_MEIxxxxx`）解压所有打包的文件。我们的代码需要能够同时访问**解压出的内置文件**和**外置的工作目录文件**。

在 `path_helper.py` 中，我们需要区分两种资源：
- **内置资源**（如 `resources/icon.ico`）：应使用 `sys._MEIPASS` 获取解压路径。
- **外置目录**（`collectors/`, `platforms/`, `config/`, `.env`）：应使用 `sys.executable` 所在目录（即用户放置 exe 的文件夹）。

因此，完善 `path_helper.py`：

```python
import sys
from pathlib import Path

def get_project_root() -> Path:
    """用户工作目录（exe所在文件夹或当前工作目录）"""
    if getattr(sys, 'frozen', False):
        return Path(sys.executable).parent
    return Path.cwd()  # 开发环境下退回当前目录

def get_builtin_root() -> Path:
    """内置资源根目录（打包时--add-data添加的文件解压位置）"""
    if getattr(sys, 'frozen', False):
        return Path(sys._MEIPASS)
    return Path.cwd()  # 开发环境下退回当前目录

PROJECT_ROOT = get_project_root()
BUILTIN_ROOT = get_builtin_root()
ICON_PATH = BUILTIN_ROOT / "resources" / "icon.ico"
```

### 4.4 动态导入外置 collectors 模块
由于 `collectors/` 是外置的 Python 文件，exe 运行时需要能够动态导入它们。实现一个导入工具：

```python
# utils/module_loader.py
import importlib.util
import sys
from pathlib import Path
from utils.path_helper import PROJECT_ROOT

def load_collector(platform_name: str):
    """从外置 collectors 目录动态加载采集器类"""
    collector_path = PROJECT_ROOT / "collectors" / f"{platform_name}.py"
    if not collector_path.exists():
        raise FileNotFoundError(f"Collector not found: {collector_path}")
    
    spec = importlib.util.spec_from_file_location(platform_name, collector_path)
    module = importlib.util.module_from_spec(spec)
    sys.modules[platform_name] = module
    spec.loader.exec_module(module)
    
    # 约定采集器类名为 {PlatformName}Collector，首字母大写驼峰
    class_name = ''.join(part.capitalize() for part in platform_name.split('_')) + "Collector"
    collector_class = getattr(module, class_name)
    return collector_class()
```

在 `controller.py` 的调度中心中，使用上述函数替代原来的静态导入。

### 4.5 外置 YAML 配置加载
`platforms/` 目录下的 YAML 文件也通过 `PROJECT_ROOT` 动态定位：

```python
def load_platform_config(platform: str):
    config_path = PROJECT_ROOT / "platforms" / f"{platform}.yaml"
    with open(config_path, 'r', encoding='utf-8') as f:
        return yaml.safe_load(f)
```
### 4.5 多次打包说明
- 每次运行 `build_exe.bat` 会重新生成 exe，**自动覆盖** `dist` 目录下的旧文件。
- 无需手动删除旧 exe，PyInstaller 会直接替换。
- 若需保留历史版本，请在打包前手动重命名或备份 `dist` 目录中的 exe。

## 5. 迁移方案（整文件夹复制）

### 5.1 用户操作
用户只需：
1. 将整个 `crawler_system/` 文件夹复制到新电脑的任意位置（如桌面、D盘、U盘）。
2. 双击文件夹内的 `多平台内容采集系统.exe`。
3. 程序自动创建运行时目录（`cookies/`, `data/` 等），读取外置配置，正常运行。

### 5.2 技术保障
- 所有路径基于 `PROJECT_ROOT`（即 exe 所在目录），因此无论文件夹被复制到何处，都能正确找到 `collectors/`, `platforms/`, `config/`, `.env`。
- 登录态（Cookie）保存在 `cookies/` 目录中，随文件夹一起迁移，用户无需重新登录。
- 历史数据（CSV）保存在 `data/` 目录中，一并迁移。

### 5.3 迁移后验证（可选）
- 打开程序，选择一个平台，输入一个已知标题，点击采集，若能正常返回数据则迁移成功。
- 若首次运行因网络问题未下载 Playwright 浏览器，程序会自动下载（约 150MB），等待即可。

## 6. 运维与平台改版应对（技术视角）

### 6.1 运维人员可修改的外置文件
| 文件 | 修改场景 | 修改后是否需要重启 |
|------|----------|--------------------|
| `.env` | 平台账号密码变更、飞书接口配置调整 | 需要重启 exe |
| `collectors/{platform}.py` | 平台登录逻辑、数据提取逻辑发生变化（极少见） | 需要重启 exe |
| `platforms/{platform}.yaml` | 平台界面改版，选择器失效（最常见） | 需要重启 exe |
| `config/exposure.yaml` | 调整曝光量数值 | 需要重启 exe |

> **重启提示**：由于 Python 模块在 exe 启动时已加载，修改外置文件后必须**完全退出程序再重新启动**才能生效。

### 6.2 平台改版应对流程（运维操作）
1. **发现问题**：采集某平台时长期无数据或报错。
2. **定位平台**：查看日志窗口或 `logs/` 目录下的最新日志，找到对应平台标识（如 `toutiao`）。
3. **修改 YAML**：用文本编辑器打开 `platforms/toutiao.yaml`，根据新的网页结构修改对应的选择器（参考 `采集器策略文档` 中的选择器更新方法）。
4. **保存文件**，确保 UTF-8 编码。
5. **重启 exe**，重新采集验证。

### 6.3 如何更新采集器代码（极少见）
如果平台改版涉及登录流程或数据提取逻辑的重大变化（例如新增了验证码、改变了翻页方式），则需要修改 `collectors/{platform}.py`。此时：
- 运维人员可请求开发人员提供新的 `.py` 文件。
- 替换 `collectors/` 目录下对应文件。
- 重启 exe 即可，无需重新打包整个程序。

### 6.4 日志与故障快照
- **日志位置**：`logs/` 目录，按天分割，保留 30 天。运维人员可查看 `error.log` 或当天日志文件定位问题。
- **故障快照**：当采集器发生未捕获异常时，程序自动保存：
  - 页面截图（`.png`）
  - DOM HTML（`.html`）
  - 异常堆栈（`.txt`）
  到 `snapshots/{platform}_{timestamp}/` 目录下，便于开发人员远程分析。

## 7. 故障排查指南（开发/运维通用）

| 现象 | 可能原因 | 检查方法 | 解决方案 |
|------|----------|----------|----------|
| 双击 exe 无反应或闪退 | 缺少 Playwright 浏览器 | 查看日志 `logs/error.log` 是否有 `BrowserType.launch` 错误 | 联网后重启，程序会自动下载；或手动执行 `playwright install` |
| 提示“找不到模块 xxx” | 外置 collectors 目录缺失或文件名错误 | 检查 `collectors/` 下是否有对应平台的 `.py` 文件 | 从备份恢复或重新生成 |
| 登录失败 | 选择器失效或账号密码错误 | 查看日志中 `login` 相关错误 | 更新 `platforms/xxx.yaml` 中的 `login` 选择器或检查 `.env` 账号密码 |
| 采集不到文章 | 列表选择器失效 | 查看日志中 `waiting for selector` 超时 | 更新 `article_list.row_selectors` |
| 某个字段为空 | 字段选择器失效 | 查看日志中 `extract field` 警告 | 更新 `field_selectors` 中对应项 |
| 程序卡死无响应 | 页面加载超时或死循环 | 查看快照目录下最新截图 | 强制结束进程，更新配置后重试 |

## 9. 方案总结

| 维度 | 实现方式 |
|------|----------|
| **路径管理** | 全局动态根目录 `PROJECT_ROOT`，所有子目录基于此拼接 |
| **打包策略** | 固定代码全打包（`main`, `gui`, `controller`, `utils` 等），仅外置 `collectors/`, `platforms/`, `config/exposure.yaml`, `.env` |
| **迁移** | 复制整个文件夹到任意位置，双击 exe 即可，无路径依赖 |
| **平台改版应对** | 运维人员直接修改外置 YAML、Python 文件或 `.env` 配置，无需重新打包 |
| **运行时数据** | 自动创建 `cookies/`, `data/`, `logs/`, `snapshots/`，随文件夹迁移 |
| **故障排查** | 日志按天分割，异常自动保存快照（截图+DOM+堆栈） |

**本方案严格遵循“完整保留原始目录结构、动态根目录、硬编码禁止”三大铁律，确保交付物可任意迁移、长期稳定运行。**
