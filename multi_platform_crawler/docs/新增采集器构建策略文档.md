# 单平台采集器构建策略文档

**版本**: v2.0  
**发布日期**: 2026-04-07  
**适用场景**: 使用 AI 编程 + Playwright MCP 工具全自动构建新平台采集器  
**目标读者**: AI 模型、开发者、系统维护人员  
**信息源**: 严格基于项目已有文档（采集器策略文档、提示词模板、总体方案、分步手册、项目情况报告）

---

## 1. 文档定位与目标

### 1.1 文档定位
本文档定义了**从零构建一个单平台采集器的完整策略**，包括：
- 前置条件与准备工作
- AI 自动化工作流（利用 Playwright MCP）
- 输入输出规范
- 选择器提取与验证规则
- 代码生成标准（基于 `template_collector.py`）
- 配置文件生成标准（基于 `template.yaml`）
- 集成测试与验收标准

### 1.2 目标
确保新平台采集器**与现有系统完全兼容**（无侵入）、**配置驱动**（选择器外置）、**健壮可靠**（多 Fallback、重试机制），且可由 AI 在最小人工输入下自动生成。

## 2. 前置条件与准备

### 2.1 必须提供的最小信息（用户输入）
用户需提供以下极简信息，AI 通过 Playwright MCP 自动完成其余分析：
示例：

```json
{
  "platform_name": "toutiao",            // 平台标识（小写+下划线）
  "display_name": "极客公园头条号",        // GUI 显示名称
  "login_url": "https://mp.toutiao.com/profile_v4/graphic/singin",
  "data_url": "https://mp.toutiao.com/profile_v4/graphic/table",
  "requires_login": true,                // 是否需要登录
  "username": "user@example.com",        // 可选，可从 .env 读取
  "password": "password"                 // 可选
}
还需要提供：登陆方式：账号密码登陆、扫码登陆；数据采集页面和导航至该页面的方式。
```

### 2.2 AI 可用工具
- **Playwright MCP**: 页面导航、快照分析、元素定位、选择器提取、登录自动化
- **Context7 MCP**: 查询 Playwright 最佳实践（如需要）
- **文件系统**: 读写 YAML、Python 文件

### 2.3 项目基线要求
- 生成的采集器必须与 `template_collector.py` 结构完全一致
- 生成的配置文件必须与 `template.yaml` 结构完全一致
- 不得修改 `collectors/` 下已有平台的任何代码
- 不得修改 `platforms/` 下已有配置

## 3. AI 自动化工作流（严格执行）

```
Step 1: 环境准备与页面启动
   ↓
Step 2: 登录页面分析（提取登录选择器）
   ↓
Step 3: 执行登录（自动或手动兜底）
   ↓
Step 4: 导航到数据后台页面
   ↓
Step 5: 文章列表结构分析（提取列表/行/字段选择器）
   ↓
Step 6: 翻页机制识别
   ↓
Step 7: 选择器测试与验证
   ↓
Step 8: 生成配置文件 (YAML)
   ↓
Step 9: 生成采集器代码 (Python)
   ↓
Step 10: 输出分析报告与集成指引
```

### 3.1 Step 1: 环境准备与页面启动
- 调用 Playwright MCP 的 `browser_navigate` 打开 `login_url`
- 等待页面加载完成：`wait_until="networkidle"`
- 截图保存：`snapshots/{platform}_login_page.png`

### 3.2 Step 2: 登录页面分析
- 调用 `browser_snapshot` 获取页面可访问树（包含元素的 `role`、`name`、`value` 等）
- 提取以下元素的多选择器（按优先级：`id` → `css` → `role` → `text` → `xpath`）：
  - **用户名输入框**
  - **密码输入框**
  - **登录按钮**
  - **验证码元素**（如有，标记为需手动处理）
- 至少为每个元素提供 **3 个不同优先级的选择器**（Fallback 机制）
- 保存截图：`snapshots/{platform}_login_analysis.png`

### 3.3 Step 3: 执行登录
- 优先尝试自动填表登录（使用提取的选择器）
- 若遇到验证码：
  - 暂停自动登录
  - 输出提示要求用户手动输入验证码（记录到日志）
  - 继续等待用户完成
- 登录成功后：
  - 调用 `browser_snapshot` 验证登录态（查找登录后独有的元素，如“发布管理”菜单）
  - 保存登录后的页面快照：`snapshots/{platform}_logged_in.png`

### 3.4 Step 4: 导航到数据后台
- 使用 `browser_navigate` 跳转到 `data_url`
- 等待文章列表加载完成：`wait_until="networkidle"`
- 截图保存：`snapshots/{platform}_data_page.png`

### 3.5 Step 5: 文章列表结构分析
- 调用 `browser_snapshot` 获取数据后台页面结构
- 识别：
  - **列表容器**（如 `table`、`div.list`、`tbody`）
  - **每一行的选择器**（如 `tr`、`.article-item`、`[data-id]`）
- 提取每个数据字段的选择器（至少 2 个备选）：

| 字段 | 说明 | 提取策略 |
|------|------|----------|
| `title` | 文章标题 | 优先从 `<a>` 或 `td[field='title']` 提取文本 |
| `url` | 文章链接 | 从标题对应的 `<a href="...">` 提取 |
| `publish_time` | 发布日期 | 从日期列提取，格式化为 `YYYY-MM-DD` |
| `read` | 阅读量 | 数字文本，去除“万”“亿”单位（需转换） |
| `like` | 点赞量 | 数字文本 |
| `comment` | 评论量 | 数字文本 |
| `forward` | 转发量 | 数字文本 |
| `collect` | 收藏量 | 数字文本 |

- 对于不存在的字段，在配置中留空或标记为 `null`，采集器自动填充 `/`

### 3.6 Step 6: 翻页机制识别
- 检查是否有“下一页”按钮、页码控件或 URL 模式
- 提取翻页选择器：
  - `next_button_selectors`: 下一页按钮选择器
  - `max_pages`: 限制翻页数（保证总文章数约 60 篇，根据每页文章数计算）
  - 若为无限滚动，记录滚动策略

### 3.7 Step 7: 选择器测试与验证
- 使用 `browser_evaluate` 测试每个选择器，确保能唯一定位元素
- 记录成功率：
  - ✅ 成功定位
  - ⚠️ 定位到多个元素（需使用 `.first` 或 `.nth(0)`）
  - ❌ 定位失败（降级使用备用选择器）
- 对于失败的选择器，尝试从 `role` 或 `text` 重新提取

### 3.8 Step 8: 生成配置文件
- 基于 `platforms/template.yaml` 模板填充
- 输出路径：`platforms/{platform_name}.yaml`
- 必须包含以下配置段：
  ```yaml
  platform: "{platform_name}"
  main_url: "{main_url}"
  login_url: "{login_url}"
  data_page_url: "{data_url}"
  username: ""   # 从 .env 或用户输入
  password: ""
  cookie_dir: "cookies"
  cookie_file: "{{cookie_dir}}/{{platform}}.enc"
  default_timeout: 30000
  element_timeout: 8000
  max_retries: 2
  user_agents: [...]   # 至少3个
  login: { username_selectors, password_selectors, login_button_selectors }
  article_list: { iframe_selectors, table_selectors, row_selectors, field_selectors, pagination }
  ```

### 3.9 Step 9: 生成采集器代码
- 基于 `collectors/template_collector.py` 模板生成
- 输出路径：`collectors/{platform_name}.py`
- 必须实现：
  - 类继承 `BaseCollector`
  - `PLATFORM_NAME = "{platform_name}"`
  - `LoginManager.is_logged_in(page)` 实现（使用配置中的验证选择器）
  - `LoginManager.perform_login(page, context)` 实现（使用配置中的登录选择器）
  - `NavigationManager.navigate_to_article_list(page)` 实现
  - `ArticleListExtractor._extract_page_articles(page)` 实现（使用配置中的字段选择器）
- **禁止修改** `template_collector.py` 中的基类方法签名

### 3.10 Step 10: 输出报告
生成 Markdown 报告，包含：
- 页面分析摘要（iframe、动态加载、反爬特征）
- 提取的选择器列表（标注优先级和测试结果）
- 截图路径
- 集成指引（如何注册到 `PlatformRegistry`）
- 注意事项（如验证码处理、特殊单位转换）

## 4. 配置文件生成规范（YAML）

### 4.1 完整模板

```yaml
# platforms/{platform}.yaml
platform: "example"
main_url: "https://example.com"
login_url: "https://example.com/login"
data_page_url: "https://example.com/dashboard"

# 认证信息（优先从 .env 读取，留空则自动填表时使用用户输入）
username: ""
password: ""

# Cookie 存储
cookie_dir: "cookies"
cookie_file: "{{cookie_dir}}/{{platform}}.enc"

# 超时配置（毫秒）
default_timeout: 30000
element_timeout: 8000
element_short_timeout: 5000

# 重试配置
max_retries: 2
retry_delay_min: 3000
retry_delay_max: 5000

# User-Agent 池
user_agents:
  - "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 ..."
  - "Mozilla/5.0 (Macintosh; Intel Mac OS X 10_15_7) ..."
  - "Mozilla/5.0 (X11; Linux x86_64) ..."

# 人类行为模拟
typing_delay_min: 50
typing_delay_max: 150
enable_stealth: true
enable_user_agent_random: true
enable_human_typing: true

# 登录模块选择器
login:
  username_selectors:
    - css: "input[name='username']"
    - id: "username"
    - role: "textbox"
      name: "请输入账号"
  password_selectors:
    - css: "input[name='password']"
    - id: "password"
    - role: "textbox"
      name: "请输入密码"
  login_button_selectors:
    - css: "button[type='submit']"
    - role: "button"
      name: "登录"
  # 可选：登录后验证元素（用于 is_logged_in 判断）
  logged_in_indicator_selectors:
    - css: ".user-avatar"
    - text: "发布管理"

# 导航模块（如果需要点击菜单）
navigation:
  menu_selectors:
    - css: ".menu-article-list"
    - role: "link"
      name: "文章管理"

# 文章列表提取模块
article_list:
  # 如果列表在 iframe 内
  iframe_selectors:
    - css: "iframe.content-frame"
    - id: "mainIframe"

  # 列表容器
  table_selectors:
    - css: "table.article-table"
    - css: ".article-list"

  # 每一行的选择器
  row_selectors:
    - css: "tbody tr"
    - css: ".article-item"

  # 各字段选择器（支持属性提取：如 "a@href" 表示取 a 标签的 href 属性）
  field_selectors:
    title: "td.title a"
    url: "td.title a@href"
    publish_time: "td.date"
    read: "td.read-count"
    like: "td.like-count"
    comment: "td.comment-count"
    forward: "td.share-count"
    collect: "td.collect-count"

  # 翻页配置
  pagination:
    next_button_selectors:
      - css: ".next-page"
      - role: "button"
        name: "下一页"
    max_pages: 3          # 限制最多翻页数（确保总文章数约60篇）
    # 可选：URL 模式翻页（如无按钮）
    # url_pattern: "https://example.com/page/{page}"
    # start_page: 1
```

### 4.2 选择器优先级规则

| 优先级 | 类型 | 示例 | 说明 |
|--------|------|------|------|
| 1 | `id` | `id="username"` | 唯一稳定，首选 |
| 2 | `css` | `input[name='account']` | 简洁高效 |
| 3 | `role` | `role="textbox" name="账号"` | 语义化，适合动态页面 |
| 4 | `text` | `text="登录"` | 文本定位，但易变 |
| 5 | `xpath` | `//div[@class='login']//input` | 灵活但脆弱，最后备选 |

- 每个元素至少提供 **3 个选择器**，按优先级降序排列
- 对于动态生成的 `id`（如 `id="u_123456"`），不使用 `id` 选择器，改用 `css` 或 `role`

## 5. 采集器代码生成规范（Python）

### 5.1 必须继承的基类结构

```python
# collectors/{platform}.py
"""
{display_name}采集器
自动生成时间: {timestamp}
"""
import asyncio
from typing import List, Dict, Tuple, Callable, Optional
from playwright.async_api import Page, BrowserContext
from .template_collector import BaseCollector
from ..utils.data_model import normalize_article
from ..utils.exposure_loader import ExposureLoader

class {PlatformName}Collector(BaseCollector):
    PLATFORM_NAME = "{platform_name}"
    
    def __init__(self):
        super().__init__(self.PLATFORM_NAME)
    
    # ========== LoginManager 必须实现的方法 ==========
    async def is_logged_in(self, page: Page) -> bool:
        """检查当前页面是否已登录"""
        # 使用配置中的 logged_in_indicator_selectors
        selectors = self.config.get("login.logged_in_indicator_selectors", [])
        for selector in selectors:
            try:
                await page.wait_for_selector(selector, timeout=3000)
                return True
            except:
                continue
        return False
    
    async def perform_login(self, page: Page, context: BrowserContext) -> bool:
        """执行自动登录（填表+点击）"""
        # 1. 定位用户名输入框
        username_selector = self._get_first_valid_selector("login.username_selectors")
        if not username_selector:
            self.logger.error("未找到用户名输入框选择器")
            return False
        await page.fill(username_selector, self.username)
        await asyncio.sleep(0.5)
        
        # 2. 定位密码输入框
        password_selector = self._get_first_valid_selector("login.password_selectors")
        if not password_selector:
            self.logger.error("未找到密码输入框选择器")
            return False
        await page.fill(password_selector, self.password)
        
        # 3. 点击登录按钮
        login_btn_selector = self._get_first_valid_selector("login.login_button_selectors")
        if not login_btn_selector:
            self.logger.error("未找到登录按钮选择器")
            return False
        await page.click(login_btn_selector)
        
        # 4. 等待登录完成
        await page.wait_for_timeout(3000)
        return await self.is_logged_in(page)
    
    # ========== NavigationManager 必须实现的方法 ==========
    async def navigate_to_article_list(self, page: Page) -> bool:
        """导航到文章列表页"""
        # 方式1: 直接访问 data_page_url
        data_url = self.config.get("data_page_url")
        if data_url:
            await page.goto(data_url, wait_until="networkidle")
            return True
        
        # 方式2: 点击导航菜单
        nav_selectors = self.config.get("navigation.menu_selectors", [])
        for selector in nav_selectors:
            try:
                await page.click(selector)
                await page.wait_for_timeout(2000)
                return True
            except:
                continue
        return False
    
    # ========== ArticleListExtractor 必须实现的方法 ==========
    async def _extract_page_articles(self, page: Page) -> List[Dict]:
        """提取当前页的所有文章（不包含翻页逻辑）"""
        articles = []
        
        # 1. 处理 iframe
        iframe_selectors = self.config.get("article_list.iframe_selectors", [])
        frame = page
        for selector in iframe_selectors:
            element = await page.query_selector(selector)
            if element:
                frame = await element.content_frame()
                if frame:
                    break
        
        # 2. 定位行元素
        row_selectors = self.config.get("article_list.row_selectors", [])
        rows = []
        for selector in row_selectors:
            rows = await frame.query_selector_all(selector)
            if rows:
                break
        
        if not rows:
            self.logger.warning("未找到文章行元素")
            return []
        
        # 3. 遍历每一行提取字段
        field_config = self.config.get("article_list.field_selectors", {})
        for row in rows:
            article = {}
            for field, selector_expr in field_config.items():
                # 支持属性提取：如 "a@href"
                if "@" in selector_expr:
                    selector, attr = selector_expr.split("@", 1)
                    element = await row.query_selector(selector)
                    value = await element.get_attribute(attr) if element else ""
                else:
                    element = await row.query_selector(selector_expr)
                    value = await element.inner_text() if element else ""
                article[field] = value.strip() if value else ""
            
            if article.get("title"):
                articles.append(article)
        
        return articles
    
    async def _go_to_next_page(self, page: Page) -> bool:
        """翻到下一页，返回是否成功"""
        next_selectors = self.config.get("article_list.pagination.next_button_selectors", [])
        for selector in next_selectors:
            try:
                next_btn = await page.query_selector(selector)
                if next_btn and await next_btn.is_enabled():
                    await next_btn.click()
                    await page.wait_for_timeout(2000)
                    return True
            except:
                continue
        return False
    
    # ========== 对外接口（无需重写，基类已实现） ==========
    # async def crawl(...) 已由 BaseCollector 实现，调用上述各管理器
```

### 5.2 必须遵守的编码约束

| 约束 | 说明 |
|------|------|
| 类名 | 必须为 `{PlatformName}Collector`，首字母大写驼峰 |
| 平台标识 | `PLATFORM_NAME` 必须与 YAML 文件名一致 |
| 方法签名 | 不可修改 `crawl` 等基类方法的参数和返回值 |
| 日志 | 使用 `self.logger.info/error/warning` |
| 重试 | 关键操作（登录、导航）使用 `@self.retry_mgr.async_retry_on_failure` |
| 选择器 Fallback | 必须实现 `_get_first_valid_selector` 辅助方法（基类已提供） |

### 5.3 数据归一化
采集器返回的原始数据字典需包含以下字段（缺失填 `""`）：
```python
{
    "title": "...",
    "url": "...",
    "publish_time": "...",
    "read": "...",
    "like": "...",
    "comment": "...",
    "forward": "...",
    "collect": "..."
}
```
调度中心会调用 `normalize_article()` 补充 `platform`、`exposure`、`recommend`（固定 `/`）等字段。

## 6. 测试验证规范

### 6.1 本地测试命令
```bash
# 单独测试采集器（使用有头浏览器便于调试）
python -c "import asyncio; from collectors.toutiao import ToutiaoCollector; c = ToutiaoCollector(); asyncio.run(c.crawl(['目标标题'], headless=False))"
```

### 6.2 验收检查清单

| 检查项 | 通过标准 |
|--------|----------|
| 配置文件存在 | `platforms/{platform}.yaml` 格式正确，无语法错误 |
| 采集器文件存在 | `collectors/{platform}.py` 无导入错误 |
| 登录功能 | 自动登录成功，Cookie 正确保存到 `cookies/` |
| 登录态验证 | `is_logged_in()` 能正确识别登录状态 |
| 导航功能 | `navigate_to_article_list()` 能到达文章列表页 |
| 列表提取 | 能正确提取至少一篇文章的标题和链接 |
| 字段映射 | 返回的字典包含所有必需字段（缺失值可为空） |
| 翻页功能 | 能正确翻页（如有翻页按钮） |
| 策略一执行 | 在测试时输入 2-3 个目标标题，能按顺序匹配并提前终止 |
| 异常处理 | 页面超时或元素缺失时不崩溃，记录日志 |

### 6.3 集成到系统
- 在 `utils/platform_registry.py` 的 `STABLE_PLATFORMS` 列表中添加平台标识
- 或通过注册表动态注册：
  ```python
  PlatformRegistry().register("toutiao", "极客公园头条号", enabled=True)
  ```

## 8. 异常处理与降级策略

| 异常情况 | AI 处理方式 |
|---------|-------------|
| 登录页无法加载 | 截图并报告，提示用户检查 URL |
| 找不到用户名输入框 | 尝试所有常见选择器（`input[type=text]`、`input[name=username]`），若仍失败则标记为“需手动登录” |
| 出现验证码 | 暂停自动登录，输出提示要求用户手动输入验证码，然后继续 |
| 数据后台无文章 | 检查是否登录态失效，尝试重新登录 |
| 翻页元素不存在 | 判断为单页或无限滚动，设置滚动采集（通过 `max_scrolls` 配置） |
| 选择器测试失败 | 降级使用 `text` 或 `xpath`，记录警告 |
| 字段值为空 | 保留空字符串，调度中心填充 `/` |
