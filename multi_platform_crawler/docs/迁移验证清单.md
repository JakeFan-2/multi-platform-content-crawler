# 迁移验证清单

本文档用于**整文件夹复制**迁移后的快速验收。详细架构与路径约定见仓库内 `.cursor/rules` 下《交付、打包、迁移、稳定运维与平台改版应对方案》及《项目总技术文档》。

---

## 一、迁移前备份（建议）

复制整个交付文件夹前，建议至少备份下列内容（避免账号、登录态与本地数据丢失）：

| 路径 | 说明 |
|------|------|
| `cookies/` | 各平台加密 Cookie / 登录态 |
| `data/` | 导出 CSV 等采集结果 |
| `.env` | 账号密码、飞书 API 等敏感配置 |
| `config/exposure.yaml` | 各平台静态曝光量等业务配置 |
| `collectors/` | 外置采集器源码（与 exe 同级，缺则无法加载平台） |

> 备份方式：压缩为 zip，或同步到网盘/另一磁盘均可。

---

## 二、迁移后操作

1. 将**整个**文件夹复制到新电脑或新路径（可重命名文件夹）。
2. 双击根目录下的主程序 exe（默认构建文件名为 `MultiPlatformCrawler.exe`，交付可重命名；或与 `main.py` 同级运行开发环境）。**新电脑首次使用**须已按包内 `首次配置说明.txt` 部署 `%LOCALAPPDATA%\ms-playwright\`。
3. 程序启动后应自动创建（若尚不存在）：`cookies/`、`data/`、`logs/`、`snapshots/`（由 `ensure_runtime_dirs()` 负责）。
4. 打开界面后查看日志区域或 `logs/` 下当日日志，**无持续报错、无闪退**即基础启动成功。

---

## 三、功能验证（**指定：ZAKER**）

按下列步骤做一次端到端冒烟，确认外置 `collectors/`、`platforms/`、`.env` 与路径解析正常：

1. 在平台列表中**勾选「ZAKER」**。
2. 在标题栏输入一篇**当前账号下能搜到的、已知存在的文章标题**（与正式使用场景一致即可）。
3. 点击**启动采集**（或等价开始按钮）。
4. **成功标准**：界面表格或日志中能**正常返回该条数据**（阅读量等字段有值或可解析），且无「采集器文件不存在」「配置文件不存在」类错误。

若能完成上述步骤，可判定**迁移与目录结构验证通过**。

## 四、常见问题自检

下列对照《交付…方案》**第 7 节 · 故障排查指南**做快速自查（现象 → 检查 → 处理）：

| 现象 | 可能原因 | 建议检查 | 处理方向 |
|------|----------|----------|----------|
| 双击 exe 无反应或闪退 / 提示 `Executable doesn't exist`、路径含 `_MEI` | 未按说明部署用户目录下的 Playwright 缓存，或版本与 exe 不匹配 | `logs/bootstrap.txt` 中 `PLAYWRIGHT_BROWSERS_PATH`；`logs/` 是否含 `BrowserType.launch` | 按包内 **首次配置说明.txt** 将 `ms-playwright` 复制到 `%LOCALAPPDATA%\`（得到 `...\AppData\Local\ms-playwright\`）；核对缓存与打包所用 Playwright 版本一致。**勿**依赖在目标机执行 `playwright install`（打包 exe 场景不适用） |
| 提示找不到模块 / 采集器 | 外置目录未随 exe 一并复制 | `collectors/` 下是否有 `zaker.py` 等同名文件 | 恢复完整文件夹；文件名与平台 ID 一致 |
| 登录失败 | 选择器失效或账号错误 | 日志中 `login` 相关报错；`.env` 中 `ZAKER_USERNAME` / `ZAKER_PASSWORD` | 改 `platforms/zaker.yaml` 登录选择器或核对 `.env` |
| 采集不到文章 | 列表行选择器失效 | 日志 `waiting for selector`、超时 | 优先改对应平台 YAML 中 `article_list.row_selectors` 等 |
| 某字段一直为空 | 字段选择器失效 | 日志中字段提取警告 | 调整 `article_list` 下各字段选择器 |
| 界面卡死 | 页面超时或逻辑阻塞 | `snapshots/` 最新截图与堆栈 | 结束进程后更新配置再试 |

## 五、自动化辅助

在项目根目录（与 `main.py` 同级）执行：

```bash
python test_migration.py
```

用于检查路径常量、稳定平台与文件配对、以及 `load_collector("zaker")` 是否成功（输出 PASS/FAIL）。

打包前可再配合：

```bash
python check_before_pack.py
```
