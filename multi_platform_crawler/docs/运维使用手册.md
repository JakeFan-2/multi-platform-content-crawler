## 1. 系统简介

本系统是一个**多平台内容数据采集工具**，支持从以下 10 个平台自动采集文章的阅读量、点赞量、评论量等运营数据：

- 微信公众号、微博、头条号、知乎、百家号、网易号、企鹅号、一点资讯、ZAKER、雪球

**核心特点**：
- **双击即用**：无需安装 Python 或任何编程环境
- **配置驱动**：所有平台的选择器、登录方式都放在外置配置文件中，页面改版**无需修改代码**，只改配置
- **自动登录**：自动保存 Cookie，下次启动无需重复登录
- **手动登录兜底**：自动登录失败时，可从界面手动触发登录并继续采集
- **手动登录采集模块**：可单独对某个平台进行有头浏览器采集，用于调试或特殊需求
- **数据导出**：支持 CSV 导出和飞书多维表格一键导入

## 2. 下载与安装

### 2.1 获取交付包
交付包是一个压缩文件（如 `crawler_system.zip`），解压后得到以下目录结构：

```
crawler_system/                     # 主文件夹（可放在任意位置）
├─ 多平台内容采集系统.exe           # 主程序（双击运行；构建默认名可能为 MultiPlatformCrawler.exe）
├─ collectors/                      # 【外置】各平台采集器 .py（须与 exe 同级完整复制）
├─ platforms/                       # 平台 YAML（重要）
│   ├─ wechat.yaml
│   ├─ toutiao.yaml
│   └─ ... (共 11 个平台文件，另可有 template 等)
├─ config/
│   └─ exposure.yaml                # 静态曝光量
├─ .env                             # 账号、飞书等敏感项（勿进 exe，由部署方配置）
├─ ms-playwright/                   # 交付包内浏览器缓存（须按说明复制到 %LOCALAPPDATA%）
├─ 首次配置说明.txt                  # 浏览器缓存部署步骤（与 main.py 行为一致）
├─ cookies/                         # 运行时生成；迁移建议保留
├─ data/                            # 运行时生成
├─ logs/                            # 运行时生成（含 bootstrap.txt 启动诊断）
└─ snapshots/                       # 运行时生成
```

### 2.2 安装步骤
1. 将 `crawler_system.zip` 解压到任意目录（如 `D:\crawler_system`）
2. **不要**将文件夹放在需要管理员权限的位置（如 `C:\Program Files`），建议放在桌面或 `D:\` 根目录
3. 解压后**必须先完成“首次运行准备”**（见下节），才能正常使用。

## 3. 首次运行准备

### 3.1 安装 Playwright 浏览器缓存（必须！）
本程序使用 Playwright 内置 Chromium 浏览器。打包后的 exe 无法通过命令行自动下载浏览器，因此**首次使用前需手动复制浏览器缓存**。

**操作步骤**：
1. 在交付包根目录下找到 `ms-playwright` 文件夹（该文件夹由开发人员提供）。
2. 将该文件夹**复制**（或剪切）到以下目标目录：
   ```
   C:\Users\你的用户名\AppData\Local\
   ```
   **提示**：在资源管理器地址栏输入 `%LOCALAPPDATA%` 回车，即可快速打开 `C:\Users\你的用户名\AppData\Local\` 文件夹。
3. 粘贴后，应存在路径：
   ```
   C:\Users\你的用户名\AppData\Local\ms-playwright\
   ```
4. 确认该文件夹大小约 150MB，内部包含 `chromium-xxxx` 等子目录。

**注意**：
- 此操作只需执行一次，后续迁移程序文件夹时，只要本机该缓存仍在，无需再次复制。
- 如果交付包中没有 `ms-playwright` 文件夹，请联系开发人员获取与当前 Playwright 版本匹配的缓存包。

### 3.2 检查网络
首次运行和采集时需要联网（用于登录平台和采集数据）。如果公司网络需要代理，请确保系统代理已配置。

## 4. 日常使用流程

### 4.1 启动程序
双击 `多平台内容采集系统.exe`，等待几秒，出现主窗口。

### 4.2 选择平台
- 在 **「1. 采集平台选择」** 区域，勾选需要采集的平台（至少选 1 个）
- 点击 **「全选」** 可一键勾选所有平台，**「全不选」** 可全部取消

### 4.3 输入目标文章标题
- 在 **「2. 标题/关键词输入」** 区域，最多输入 **5 个**标题（每个标题一行输入框）
- 支持直接粘贴（Ctrl+V 或右键粘贴）
- **特殊规则**：若勾选了 **微博** 或 **头条号**，且**某一标题超过 30 字**，系统会在**该标题输入框下方**显示对应的**专属关键词**输入框，须填写关键词后再采集（用于列表页模糊匹配）

### 4.4 开始自动采集
- 点击 **「启动采集」** 按钮
- 系统按队列依次处理各平台（无头浏览器）
- 在 **「5. 实时日志」** 区域查看进度
- 若某平台在自动任务中登录失败，会进入 **「4. 手动登录平台」** 列表；选中后点击 **「手动登录采集」**，会弹出浏览器完成登录并采集，结果写入**同一张主表**（与「手动登录采集模块」走同一套后台逻辑）。自动任务中该平台可能已被跳过，如需补采可再次 **「启动采集」** 或直接使用「手动登录采集模块」

### 4.5 手动登录采集模块（独立分组）
界面分组标题为 **「手动登录采集模块」**（可折叠）。列表中包含 **全部 11 个稳定平台**，其中微信公众号、企鹅号、雪球号**置顶显示**，便于优先处理扫码类平台。

**使用步骤**：
1. 在**主界面标题区至少填写 1 个目标标题**（与自动采集共用同一批标题框）
2. 在模块列表中**单选**一个平台
3. 点击 **「手动登录并采集」**
4. 系统将加载 exe 同级外置 `collectors/<平台>.py`，以**有头浏览器**执行登录与采集（与自动任务同源逻辑，**无需**单独安装 Python）
5. 采集过程中数据会写入**同一张主数据表**

**注意事项**：
- **勿在采集中重复点击**「手动登录采集」或「手动登录并采集」：界面会在流程进行中禁用两枚按钮；调度器侧也会拒绝重叠的手动有头任务。请等当前浏览器流程结束、按钮恢复后再操作
- 若此时**整次自动采集任务仍在进行**，调度器会先**暂停队列**，待本操作结束后再**自动恢复**（无需手工处理）
- 浏览器窗口请保持至采集结束；若登录或页面超时，可按日志提示后重试
- 仍使用本机 Cookie 目录下已保存的登录态；若已失效，请在弹出浏览器中重新完成登录

### 4.6 查看结果
- 数据实时显示在 **「6. 采集数据结果」** 表格中
- 未匹配标题显示在表格下方列表中

### 4.7 停止/暂停与再次采集
- **暂停**：暂停调度；点 **「恢复」** 继续
- **停止**：终止当前任务（已进表的数据保留）
- 一次任务结束后，**可直接再次点击「启动采集」** 开始新任务，**不必**重启整个程序

## 5. 配置文件说明与维护

### 5.1 曝光量配置文件（`config/exposure.yaml`）
**位置**：`crawler_system/config/exposure.yaml`  
**作用**：为每个平台设置固定的“曝光量”数值（单位：万），该值会覆盖采集到的原始曝光量（如果平台本身不提供曝光量，则直接填充此值）。

**格式示例**：
```yaml
exposure:
  wechat: "800"   # 微信公众号 800 万
  toutiao: "600"           # 头条号 600 万
  weibo: "500"             # 微博 500 万
  # ... 其他平台
```

**如何修改**：
1. 用记事本或任何文本编辑器打开该文件
2. 找到对应平台的标识（如 `weibo`），修改引号内的数字
3. 保存文件后 **重启程序** 再采集（曝光量配置在进程内单例加载，运行中修改文件不会自动重载）

**注意**：
- 不要修改平台标识（冒号左边的名字）
- 数值可以是整数或小数，不需要带“万”字
- 如果不希望注入曝光量，可以删除该平台的行，系统会填充 `/`

### 5.2 平台配置文件（`platforms/*.yaml`）
**位置**：`crawler_system/platforms/` 目录下，每个平台一个 `.yaml` 文件  
**作用**：定义平台的登录方式、文章列表选择器、字段提取规则等。**运维人员通常不需要修改**，只有在平台页面改版时才需要更新选择器（详见第6节）。

**重要提醒**：
- **不要随意修改**这些文件，除非平台改版导致采集失败
- 修改前请备份原文件
- 修改后保存，重启程序生效

### 5.3 页面定位/判断专用配置
【初次使用必配】页面判断专用账号名称配置：需严格按照账号实际页面的信息填写账号名称；10 个平台的配置信息需独立区分，禁止重叠混用，避免采集定位混乱。

**配置位置**：在 `.env` 文件的页面定位/判断专用配置模块中填写

**配置项**：
- `{PLATFORM}_PLATFORM_NAME`：平台名称（页面定位用）
- `{PLATFORM}_ACCOUNT_NICKNAME`：账号昵称（侧边栏/页面判断用，需与账号页面实际显示的名称完全一致）

**注意事项**：10个平台的配置信息需独立区分，禁止重叠混用，避免采集定位混乱

### 5.4 .env 文件配置提醒（新用户必看）
**配置步骤**：
1. 用户下载代码后，在项目目录中找到 `.env.example` 文件（这是模板文件，公开无害）
2. 在同一目录下，新建一个空白文件，命名为 `.env`
3. 打开 `.env.example`，全选复制所有内容
4. 粘贴到新建的 `.env` 文件里
5. 用户把自己的隐私数据（如账号密码、API 密钥等）填进 `.env` 即可

**运行机制**：程序运行时，会自动读取 `.env` 文件，完美运行

## 6. 平台改版应对流程（重要！）

### 6.1 什么时候需要应对改版？
当你发现某个平台**一直采集不到数据**或**登录失败**，且日志中出现类似 `Timeout waiting for selector` 或 `Element not found` 的错误时，很可能是平台更新了页面结构，导致原有的选择器失效。

### 6.2 解决方案概述
**不需要修改代码，也不需要重新打包**。只需：
1. 分析目标平台的新页面结构，获取新的选择器
2. 修改对应平台的 `.yaml` 配置文件中的选择器
3. 保存文件，重启程序

### 6.3 使用 AI 编程工具辅助分析（推荐）
为了快速准确地获取新选择器，建议使用 **AI 编程工具 + Playwright MCP** 进行半自动分析。推荐工具：
- **Cursor**（内置 AI + 浏览器控制）
- **VS Code + Continue 插件 + Playwright MCP**

**操作步骤**（以头条号为例）：

1. **打开 AI 编程工具**，创建一个新的对话窗口。
2. **提供以下提示词**（可直接复制）：
   ```
   我需要分析头条号的文章列表页，提取新的选择器。
   请使用 Playwright MCP 工具：
   1. 打开 https://mp.toutiao.com/profile_v4/graphic/table
   2. 登录后（如需手动登录请提示我）
   3. 获取页面快照，分析文章列表的结构
   4. 提取以下元素的选择器（至少提供2个备选）：
      - 文章标题（及链接）
      - 发布日期
      - 阅读量
      - 点赞量
      - 评论量
   5. 输出 YAML 格式的选择器配置，可以直接替换到 platforms/toutiao.yaml 中
   ```
3. **AI 会自动启动浏览器**，可能需要你手动登录（输入验证码等）。登录后，AI 会分析页面并给出新的选择器。
4. **将 AI 输出的选择器内容**，替换到 `platforms/toutiao.yaml` 文件中对应的部分（通常只修改 `article_list.field_selectors` 和 `login` 部分）。
5. **保存文件**，重启采集程序，测试是否恢复正常。

### 6.4 手动获取选择器的方法（备选）
如果没有 AI 工具，也可以手动用浏览器开发者工具获取：

1. 用 Chrome 浏览器打开目标平台的后台页面（需先登录）
2. 按 `F12` 打开开发者工具
3. 点击左上角的 **选择元素** 图标（箭头）
4. 点击页面上的文章标题，在 `Elements` 面板中会高亮对应的 HTML
5. 右键该元素 → **Copy** → **Copy selector** 或 **Copy XPath**
6. 将复制的内容替换到 YAML 文件中对应的选择器

**示例**：  
原来 YAML 中的标题选择器是：
```yaml
title: "td.title a"
```
如果新的页面变成：
```html
<div class="article-item">
  <a class="title-link" href="...">文章标题</a>
</div>
```
则应修改为：
```yaml
title: ".article-item .title-link"
```

### 6.5 修改后验证
- 保存 YAML 文件
- 重启采集程序
- 单独勾选该平台，输入一个已知的文章标题，点击“启动采集”
- 如果能正常返回数据，说明修改成功

## 7. 数据导出与飞书导入

### 7.1 导出 CSV
- 采集完成后，点击 **“导出CSV (UTF-8 BOM)”** 按钮
- 文件自动保存在 `crawler_system/data/` 目录下，文件名格式：`export_YYYYMMDD_HHMMSS.csv`
- 该文件可用 Excel 直接打开（中文不会乱码）

### 7.2 导入飞书多维表格（可选）
- **前提**：飞书应用凭证由开发预先写入 `.env`（运维一般无需改代码）
- **使用**：**「导入飞书」** 在程序启动后即可点击，将**当前表格中已有数据**批量导入飞书（不必等待某次采集完全结束）；导入在后台线程执行，完成后会有提示

## 8. 系统迁移（换电脑/换目录）

### 8.1 迁移步骤（极简）
1. **复制整个 `crawler_system` 文件夹**到新电脑的任意位置（如 `D:\crawler_system`）
2. **在新电脑上完成首次运行准备**（即复制 `ms-playwright` 文件夹到 `%LOCALAPPDATA%`，参见第3节）
3. **双击 exe** 即可使用，之前保存的登录状态（cookies）和数据都会保留

### 8.2 注意事项
- 如果迁移后某个平台提示登录失效，是因为 Cookie 中的设备信息与当前环境不匹配，**重新手动登录一次**即可（系统会自动保存新的 Cookie）
- 日志文件和快照目录可以定期清理（不影响使用）

## 9. 常见问题与故障处理

### 9.1 程序无法启动
| 现象 | 可能原因 | 解决方法 |
|------|----------|----------|
| 双击 exe 没反应 | 缺少 VC++ 运行库 | 安装 [VC++ 2015-2022 Redistributable](https://aka.ms/vs/17/release/vc_redist.x64.exe) |
| 提示“无法定位程序输入点” | Windows 版本过旧 | 升级 Windows 10 或更高版本 |
| 提示“Executable doesn't exist” | 未复制浏览器缓存 | 按第3节复制 `ms-playwright` 到 `%LOCALAPPDATA%` |

### 9.2 采集时一直提示“登录失败”
- **自动登录失败**：平台可能更新了登录页，需要更新 `login` 部分的选择器（参见第6节）
- **账号密码错误**：检查账号密码是否正确（如需修改，联系开发人员更新 `.env` 文件或 YAML 中的 `username/password`）

### 9.3 某个平台一直采集不到文章
- 原因：平台改版，选择器失效
- 解决：按照第6节流程，更新该平台的 YAML 配置文件

### 9.4 手动登录采集弹出浏览器后无反应
- 在弹出窗口中**手动完成登录**（扫码或验证码等）
- 登录完成前勿关闭浏览器；若页面或会话超时，查看日志后**重新点击**「手动登录采集」或「手动登录并采集」
- **「手动登录并采集」** 须先在主界面填至少 **1 条标题**，否则会提示无法开始
- 一次有头流程未结束时**不要连点**上述按钮；若日志出现「已有手动采集任务进行中」，请等待当前流程结束后再试

### 9.5 采集速度很慢
- 网络慢或平台响应慢，属于正常现象
- 需要观察浏览器时，可使用 **「4. 手动登录平台」** 或 **「手动登录采集模块」** 触发**有头**采集；当前工具栏**无**单独的「调试模式」复选框

### 9.6 导出的 CSV 用 Excel 打开乱码
- 确保使用 **“导出CSV (UTF-8 BOM)”** 按钮，不要手动复制表格内容
- 如果仍然乱码，用记事本打开 CSV，另存为 ANSI 编码（不推荐）

### 9.7 日志文件太大
- 系统自动按天分割日志，保留 30 天，无需手动清理
- 如需手动删除，可以删除 `logs/` 目录下的旧 `.log` 文件

### 9.8 飞书导入报错 `No module named 'aiohttp'`
- 该问题已在最新版本中修复，如果仍然出现，请联系开发人员检查打包环境是否正确包含了 `aiohttp` 依赖

## 10. 注意事项

1. **不要删除或移动 `platforms/`、`config/` 文件夹**，否则程序无法启动
2. **不要手动修改 `cookies/` 目录下的文件**，它们是加密的登录凭证
3. **不要同时运行两个 exe 实例**，可能导致 Cookie 冲突
4. **首次使用务必按第3节复制浏览器缓存**，否则无法启动浏览器
5. **平台改版时只修改 YAML 配置文件**，不要尝试修改 exe 或 Python 代码
6. **定期备份整个 `crawler_system` 文件夹**（建议每月一次），防止数据丢失
7. **如果遇到无法解决的问题**，请收集 `logs/` 目录下的最新日志文件，联系开发人员分析
