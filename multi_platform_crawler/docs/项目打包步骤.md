### 📦 多平台内容采集系统打包步骤（完整版）

#### 🔹 环境说明
- **项目根目录**: `D:\PythonProject5\multi_platform_crawler`
- **打包产物**: `dist\MultiPlatformCrawler.exe`
- **以下命令需在项目根目录下执行**

---

### 1️⃣ 删除旧版可执行文件（避免冲突）

**PowerShell 与 CMD 通用**：
```cmd
del /f "D:\PythonProject5\multi_platform_crawler\dist\MultiPlatformCrawler.exe"
```
> 若文件不存在会提示“找不到文件”，忽略即可。
> 也可以选择手动删除。

---

### 2️⃣ 进入项目目录并激活虚拟环境

#### 🔸 选项 A：使用 PowerShell

```powershell
cd D:\PythonProject5\multi_platform_crawler
.\.venv\Scripts\Activate.ps1
```
> 若提示“无法加载文件”，请先执行：`Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass`

#### 🔸 选项 B：使用命令提示符（CMD）

```cmd
cd /d D:\PythonProject5\multi_platform_crawler
.venv\Scripts\activate.bat
```

> 若项目目录下没有 `.venv` 文件夹，需先创建并安装依赖（命令略，不属于本次打包步骤范围）。

---

### 3️⃣ 打包前检查（建议必做）

**PowerShell / CMD 通用**（需已激活虚拟环境）：
```cmd
python check_before_pack.py
```
- 根据脚本输出的 `[PASS]` / `[FAIL]` / `[WARN]` 处理。
- **有 `[FAIL]` 时请先修复再继续打包**。

---

### 4️⃣ 执行打包

#### 🔸 推荐方式：运行现成脚本 `build_exe.bat`

**PowerShell 中执行**：
```powershell
.\build_exe.bat
```

**CMD 中执行**：
```cmd
build_exe.bat
```

> 脚本会自动调用 `.venv\Scripts\pyinstaller.exe`（若不存在则回退到系统 pyinstaller），生成单文件无控制台窗口的 exe。

---

### 5️⃣ 确认产物位置

打包成功后，可执行文件位于：
```
D:\PythonProject5\multi_platform_crawler\dist\MultiPlatformCrawler.exe
```
中间文件在 `build\` 目录，可按需保留或删除。

---

### 6️⃣ 交付目录要求（简要提示）

- 交付时需将 `MultiPlatformCrawler.exe` 与以下**外置目录/文件**放在同一文件夹：
  - `collectors\`
  - `platforms\`（若未完全打入包中）
  - `config\`（若需外置配置）
  - 首次运行会自动生成 `cookies\`、`data\`、`logs\` 等。
- 外置 `collectors\` **必须与 exe 同级**，否则程序启动后无采集器可用。

---

### 7️⃣ Playwright 浏览器问题（仅当目标机器报缺浏览器时处理）

打包已通过 `--collect-all=playwright` 包含相关文件。若目标机仍提示缺少 Chromium，请在**装有 Playwright 的开发环境**执行：
```cmd
playwright install chromium
```
并按《交付与运维文档》设置 `PLAYWRIGHT_BROWSERS_PATH` 环境变量指向已下载的浏览器目录。
